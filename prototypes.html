Card Battlers

Inventory management was beginning to become important to me because one of the most fun mechanics of a game (IMO) is the loot system. And you can't have a good loot system without proper inventory management!

Scriptable Objects!
This was a project that I devised to teach myself about scriptable objects in Unity as well as trying to learn some sort of solution in regards to the inventory management problem. Using a database made from a scriptable object, I was able to save all of my card's data, their images, descriptions, stats, etc. in a single place and have it be easily changable any time.

My biggest failure here was in accidentally changing the database card values as opposed to the actually owned card, leading to weird stat issues. Since then I have learned to store those items in a new list and in essence only refer to these newly cloned list items. I learned that SO's should only be used to store data, and it should be avoided to manipulate that data at runtime.
